## Historia 1: El Videoclub de Martin

Martin acaba de heredar el videoclub de su abuelo en el barrio de Caballito. El negocio esta algo desactualizado, pero tiene una base de datos con toda la informacion de las peliculas, los socios y los alquileres. Martin quiere modernizar el negocio y necesita tu ayuda para analizar los datos y tomar decisiones. Que peliculas son las mas populares? Que socios estan inactivos? Cuanto factura el videoclub por mes? Ponete en el lugar de Martin y usa SQL para responder estas preguntas.

### Esquema de la Base de Datos

```sql
CREATE TABLE peliculas (
    id SERIAL PRIMARY KEY,
    titulo TEXT NOT NULL,
    genero TEXT NOT NULL,
    anio INTEGER NOT NULL,
    director TEXT NOT NULL,
    precio_alquiler DECIMAL(5,2) NOT NULL
);

CREATE TABLE socios (
    id SERIAL PRIMARY KEY,
    nombre TEXT NOT NULL,
    email TEXT NOT NULL,
    fecha_registro DATE NOT NULL,
    activo BOOLEAN DEFAULT true
);

CREATE TABLE alquileres (
    id SERIAL PRIMARY KEY,
    socio_id INTEGER REFERENCES socios(id),
    pelicula_id INTEGER REFERENCES peliculas(id),
    fecha_alquiler DATE NOT NULL,
    fecha_devolucion DATE,
    devuelto BOOLEAN DEFAULT false
);
```

    <CodePlayground language="sql" title="Tu consulta" height="150px" initialCode={`-- Filtra alquileres no devueltos donde la fecha_alquiler sea anterior a hace 30 dias\n-- Pista: usa CURRENT_DATE - INTERVAL '30 days'\n`} />
  </TabItem>
  <TabItem value="solucion" label="💡 Solución">

## Historia 2: La Pizzeria de los Hermanos Lopez

Los hermanos Lopez abrieron una pizzeria en Palermo hace 2 anios. El negocio crecio tanto que necesitan organizar mejor sus datos. Tienen informacion sobre el menu, los pedidos, los repartidores y los clientes. Necesitan analizar que pizzas son las mas vendidas, cuanto facturan por dia, y que repartidores son los mas eficientes. Tu SQL va a ayudarlos a tomar mejores decisiones!

### Esquema de la Base de Datos

```sql
CREATE TABLE menu (
    id SERIAL PRIMARY KEY,
    nombre TEXT NOT NULL,
    categoria TEXT NOT NULL,  -- 'pizza', 'empanada', 'bebida', 'postre'
    precio DECIMAL(7,2) NOT NULL,
    disponible BOOLEAN DEFAULT true
);

CREATE TABLE clientes (
    id SERIAL PRIMARY KEY,
    nombre TEXT NOT NULL,
    telefono TEXT NOT NULL,
    direccion TEXT NOT NULL,
    barrio TEXT NOT NULL
);

CREATE TABLE repartidores (
    id SERIAL PRIMARY KEY,
    nombre TEXT NOT NULL,
    vehiculo TEXT NOT NULL,  -- 'moto', 'bicicleta', 'auto'
    activo BOOLEAN DEFAULT true
);

CREATE TABLE pedidos (
    id SERIAL PRIMARY KEY,
    cliente_id INTEGER REFERENCES clientes(id),
    repartidor_id INTEGER REFERENCES repartidores(id),
    fecha DATE NOT NULL,
    total DECIMAL(8,2) NOT NULL,
    estado TEXT NOT NULL  -- 'pendiente', 'en_camino', 'entregado', 'cancelado'
);

CREATE TABLE detalle_pedidos (
    id SERIAL PRIMARY KEY,
    pedido_id INTEGER REFERENCES pedidos(id),
    menu_id INTEGER REFERENCES menu(id),
    cantidad INTEGER NOT NULL,
    subtotal DECIMAL(8,2) NOT NULL
);
```

## Historia 3: La Universidad Nacional de Codigo

La Universidad Nacional de Codigo (UNC) necesita modernizar su sistema de gestion academica. Tienen datos de estudiantes, materias, profesores e inscripciones. El rector te pide ayuda para generar reportes: que materias tienen mas inscriptos? Que profesores dictan mas materias? Que estudiantes estan en riesgo de quedar libres? Con tus consultas SQL, vas a ayudar a la universidad a tomar mejores decisiones academicas.

### Esquema de la Base de Datos

```sql
CREATE TABLE estudiantes (
    id SERIAL PRIMARY KEY,
    nombre TEXT NOT NULL,
    legajo TEXT UNIQUE NOT NULL,
    carrera TEXT NOT NULL,
    anio_ingreso INTEGER NOT NULL
);

CREATE TABLE profesores (
    id SERIAL PRIMARY KEY,
    nombre TEXT NOT NULL,
    departamento TEXT NOT NULL,
    antiguedad INTEGER NOT NULL  -- anios
);

CREATE TABLE materias (
    id SERIAL PRIMARY KEY,
    nombre TEXT NOT NULL,
    departamento TEXT NOT NULL,
    creditos INTEGER NOT NULL,
    cuatrimestre INTEGER NOT NULL  -- 1 o 2
);

CREATE TABLE inscripciones (
    id SERIAL PRIMARY KEY,
    estudiante_id INTEGER REFERENCES estudiantes(id),
    materia_id INTEGER REFERENCES materias(id),
    profesor_id INTEGER REFERENCES profesores(id),
    nota DECIMAL(4,2),  -- NULL si aun no tiene nota
    estado TEXT NOT NULL  -- 'cursando', 'aprobado', 'desaprobado', 'libre'
);
```
